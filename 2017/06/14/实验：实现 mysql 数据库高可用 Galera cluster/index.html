实验：实现 mysql 数据库高可用 Galera cluster

实验： 实现 Galera cluster

首先准备三台主机，分别为 192.168.30.7    192.168.30.17     192.168.30.27

首先安装包 ，这个包不是版本自带的数据库包，是专门的 galera-server 数据库包，一般可以在互联网查找比较快的yum 源，如清华大学的就可以。
如： 清华yum 源 ： 

     [mariadb]
 name = MariaDB
baseurl = https://mirrors.tuna.tsinghua.edu.cn/mariadb/mariadb-5.5.60/yum/centos7-amd64/
 gpgcheck=0

在每台主机上都要安装相应的包。
yum install MariaDB-Galera-server

安装完后再配置文件进行修改
Vim /etc/my.cnf.d/server.cnf
在里面添加内容
wsrep_provider = /usr/lib64/galera/libgalera_smm.so
wsrep_cluster_address=”gcomm://192.168.30.7，192.168.30.17，192.168.30.27”
binlog_format=row

修改完后启动服务：
启动服务时要在三台主机中挑选一个主机执行启动命令 ：
/etc/init.d/mysql start –wsrep-new-cluster
然后在剩下两个主机上执行启动命令：
service mysql start

启动后可以拿变量来查看下一些相关的状态信息：
show status like ‘wsrep%’;

到此就实现了 galera cluster 数据库的主主拓扑结构，三台主机互相同步，并且都具有修改等操作的能力。
