实验：centos 7.4 实现二进制安装mariadb-10.2.15-linux-x86_64.tar.gz mysql 数据库

1，检查相应的坏境
查看下防火墙iptables和selinux 是否关闭，没关闭先关闭。
再查看下原来主机上是否已有 mariadb 数据库，如果有先卸载在进行安装。

2,下载对应的二进制文件


3，准备用户账号，mysql，顺便指定数据库家目录，系统默认是 /var/lib/mysql/ 目录下，但是在生产中一般都是自己指定，现在我们指定在 /data/mysqldb/ 目录下。
useradd -r -d /data/mysqldb -s /sbin/nologin mysql

接下来要创建一个安装路径 ，这个路径将来要把解压缩包的内容存放在那，但是这个路径在压缩包里已经有定义了，所以我们要放进人家对应的目录下 /usr/local/ .
解压缩 tar xvf mariadb-10.2.15-linux-x86_64.tar.gz -C /usr/local/

解压缩完会在对应目录下生成一个带版本号的文件，这是不符合条件的，所以要么改名，要么使用软连接。当然，使用软连接更方便点。 

创建软连接      ln -s mariadb-10.2.15-linux-x86_64/   mysql

但是会发现 cd 进到 mysql 这个目录下，里面的文件的所属组跟所有者都是一些没有用户的 UID 跟 GID 。所以我们要把所有者跟所属组都改为 root 。
chown -R mysql:mysql mysql/

接下来准备一些变量，因为在这个目录下，存放着一些主程序工具，为方便实用，放进 PATH 变量里。
echo PATH=/usr/local/mysql/bin:$PATH > /etc/profile.d/mysql.sh
生效一下： . /etc/profile.d/mysql.sh

接下来就要规划用户的数据库放在哪，还考虑到将来这里的数据会很大，所以建议用逻辑卷来做家目录，以方便以后空间不足还可以扩充。
创建逻辑卷
先得加块大硬盘，比如： sdb 。 加上扫描一下
echo ‘ - - - ‘ > /sys/class/scsi_host/host2/scan
然后把整块硬盘都做成逻辑卷。
pvcreate /dev/sdb 生成物理卷
vgcreate vg0 /dev/sdb 加入卷组
lvcreate -n lv_mysql -l 100%FREE vg0 创建逻辑卷
接下来创建文件系统。
mkfs.xfs /dev/vg0/lv_mysql
然后创建一个目录，把逻辑卷挂载上去，要想永久挂载，还要写进文件。
mkdir /data/
echo “ /dev/vg0/lv_mysql /data/ xfs defaults 0 0 “ >> /etc/fstab
然后挂载 mount -a

接下来创建用户存放数据的家目录，/data/mysqldb/
mkdir /data/mysqldb/
并修改此用户的权限信息。
chown mysql.mysql /data/mysqldb
chmod 770 /data/mysqldb
这样就把家目录准备好了。
一定要注意 此目录不要跟 /usr/local/mysql/ 这个目录搞混了。
/data/mysqldb/ 这是存放将来用户数据的数据库。
/usr/local/mysql/ 这是存放二进制程序的目录

接下来我们还要在用户数据mysqldb/库 /data/ 中生成一些系统数据库，要用一个脚本来生成。
执行这个脚本时，一定要 cd 进入到 /usr/local/mysql/ 目录下
scripts/mysql_install_db –datadir=/data/mysqldb –user=mysql

运行成功后就会在 /data/mysqldb/ 目录下生成一些文件。

接下来修改配置文件，来告诉二进制程序用户存放数据的数据库路径，mysql 数据库的主配置文件在 /etc/my.cnf

修改数据库要参照 mysql/support-files/ 这个目录下的一些配置文件，根据生产需求拷贝想用的文件去覆盖主配置文件。
cp support-files/my-huge.cnf /etc/my.cnf
然后进行修改 vim /etc/my.cnf
在mysqld 语句块中添加路径 datadir =/data/mysqldb
这样数据库的配置文件就准备好了。

在准备启动脚本
在 /usr/local/mysql/support-files/ 目录下二进制程序已经有了相关的脚本，我们拷贝一份就可以了。
cd /usr/local/mysql/
cp support-files/mysql.server /etc/init.d/mysqld

加到启动服务中去
chkconfig –add mysqld
接下来就可以启动服务了
systemctl start mysqld
启动成功后查看下端口打开没 。

到此二进制安装就成功了。
就可以进行连接了。
但是直接连接会存在安全问题。所以把安全脚本跑一边就可以了。
mysql_secure_installation
跑脚本时输入相应信息就可以了。
